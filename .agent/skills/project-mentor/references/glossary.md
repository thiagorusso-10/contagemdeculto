# Glossário do Projeto

Aqui estão explicações diretas para as palavras "técnicas" que você pode ver neste projeto.

## Termos Comuns

### Component (Componente)
**Analogia**: Um bloco de Lego.
**Definição**: Uma peça reutilizável da interface. Botões, cartões e cabeçalhos são todos componentes. Nós construímos o aplicativo encaixando esses blocos.

### Props (Propriedades)
**Analogia**: Configurações ou Ajustes.
**Definição**: Informação que passamos *para dentro* de um componente para mudar como ele parece ou se comporta. Como dizer a um componente "Botão" para ser "Azul" ou "Vermelho".

### State (Estado)
**Analogia**: Uma memória ou painel de controle.
**Definição**: Dados que mudam com o tempo. Se um usuário digita em uma caixa, esse texto está no "Estado". Se o aplicativo lembra que você está logado, isso é "Estado".

### Hook (Gancho)
**Analogia**: Uma ferramenta especial do cinto de utilidades.
**Definição**: Funções que começam com `use` (como `useState`). Elas permitem que nossos componentes façam coisas inteligentes, como lembrar dados ou verificar mensagens (requisições de rede).

### API / Supabase
**Analogia**: O Garçom.
**Definição**: O mensageiro que anota seu pedido (requisição de dados) leva para a cozinha (banco de dados) e traz a comida (dados) de volta para sua mesa (aplicativo).
